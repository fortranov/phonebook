# Телефонный справочник — документация API

Справочник предоставляет HTTP API для поиска сотрудников по ФИО и получения их телефонных номеров.

---

## Базовый URL

```
http://<host>/api.php
```

---

## Эндпоинты

### `GET /api.php` — поиск по справочнику

| Параметр | Тип    | Обязательный | По умолчанию | Описание |
|----------|--------|:------------:|:------------:|----------|
| `action` | string | нет          | `search`     | Действие. В текущей версии единственное значение — `search` |
| `q`      | string | нет          | *(пусто)*    | Поисковый запрос. Ищет по всем полям записи. Поддерживает несколько слов через пробел (AND-логика). При пустом значении возвращает все записи |
| `limit`  | int    | нет          | `50`         | Максимальное число записей в ответе. Допустимый диапазон: 1–200 |
| `offset` | int    | нет          | `0`          | Смещение для пагинации |

#### Особенности поиска

- Поиск **регистронезависимый** и работает по **всем полям** (организация, ФИО, должность, телефоны, адрес, комментарий).
- Если в `q` передано несколько слов, возвращаются только записи, содержащие **все** слова одновременно.
- Пример: `q=Иванов Сергей` найдёт «Иванов Сергей Петрович», но не «Иванов Пётр».

---

## Формат ответа

### Успешный ответ (`200 OK`)

```json
{
  "success": true,
  "total":   25,
  "offset":  0,
  "limit":   50,
  "count":   25,
  "records": [
    {
      "организация": "ФСБ России",
      "фио":         "Иванов Сергей Петрович",
      "должность":   "полковник",
      "служебный":   "78-90-12",
      "городской":   "8(495)555-12-34",
      "мобильный":   "8-906-123-45-67",
      "адрес":       "г. Москва",
      "комментарий": "Руководитель отдела"
    }
  ]
}
```

#### Поля ответа верхнего уровня

| Поле      | Тип     | Описание |
|-----------|---------|----------|
| `success` | bool    | `true` при успешном выполнении запроса |
| `total`   | int     | Общее число записей, соответствующих запросу (без учёта пагинации) |
| `offset`  | int     | Смещение, переданное в запросе |
| `limit`   | int     | Лимит, переданный в запросе |
| `count`   | int     | Число записей в текущей странице (`records.length`) |
| `records` | array   | Массив найденных записей |

#### Поля объекта записи

| Поле           | Тип    | Описание |
|----------------|--------|----------|
| `организация`  | string | Название организации или подразделения |
| `фио`          | string | Фамилия Имя Отчество |
| `должность`    | string | Должность сотрудника |
| `служебный`    | string | Служебный телефон. Формат: `NN-NN-NN` (например, `78-90-12`). Может быть пустой строкой |
| `городской`    | string | Городской телефон. Формат: `8(XXX)NNN-NN-NN` или `(XXX)NNN-NN-NN`. Может быть пустой строкой |
| `мобильный`    | string | Мобильный телефон. Может быть пустой строкой |
| `адрес`        | string | Адрес. Может быть пустой строкой |
| `комментарий`  | string | Произвольный комментарий. Может быть пустой строкой |

### Ответ при ошибке (`400 Bad Request`)

```json
{
  "success": false,
  "error":   "Неизвестное действие: foo"
}
```

---

## Пагинация

При большом числе записей используйте параметры `limit` и `offset`.

Признак наличия следующей страницы: `offset + count < total`.

```
Страница 1: offset=0,  limit=50
Страница 2: offset=50, limit=50
...
```

## Примеры запросов

### Найти по фамилии

```
GET /api.php?q=Иванов
```

### Найти по имени и фамилии

```
GET /api.php?q=Сергей Петрович
```

### Получить всех сотрудников (первые 100)

```
GET /api.php?limit=100
```

### Постраничный перебор — вторая страница по 20 записей

```
GET /api.php?q=полковник&limit=20&offset=20
```

## Пример интеграции (Python)

```python
import requests

BASE_URL = "http://<host>/api.php"

def find_phones(full_name: str) -> list[dict]:
    """Вернуть список записей, совпадающих с запросом full_name."""
    response = requests.get(BASE_URL, params={"q": full_name, "limit": 50})
    response.raise_for_status()
    data = response.json()
    if not data["success"]:
        raise RuntimeError(data.get("error", "Неизвестная ошибка"))
    return data["records"]

# Использование
records = find_phones("Иванов Сергей")
for rec in records:
    print(rec["фио"], "— служебный:", rec["служебный"], "| городской:", rec["городской"])
```

## Ограничения

- Метод `POST` не поддерживается — только `GET`.
- Аутентификация отсутствует. При необходимости ограничьте доступ на уровне веб-сервера (IP-whitelist, Basic Auth и т.п.).
- Максимальный `limit` — 200 записей за запрос.
- Данные отдаются в кодировке **UTF-8**.
